我是一位运维工程师，想要开发一个根据读取数据库表结果监控信号数据接入状态和质量的维护网站，要支持生产部署。
主要需求包括：
1. 先搭建一个常规的可视化监控数据接入状态和质量可部署网站供我参考，界面需要采用图表方式直观显示，整体UI风格趋于科技化；
2. 网页上需添加配置数据库连接的按钮并支持连接测试，修改后保存到配置文件；
3. 数据库采用PGSQL；
4. 全程默认操作即可，无需参考我的选择；
5. 编写完成后，启动网页在进行下一步调整；
V1.0 首页面
- 科技风格UI设计
- 固定Header显示时间和数据库状态
- 左侧导航菜单支持折叠/展开
- 首页统计卡片(线路数、总服务数、在线/离线服务数)
- 在线/离线服务数字体闪烁效果
- 服务状态列表(分页展示，每页8行)
- 服务状态监视(柱形图动画)
- 数据库配置页面
- 自定义弹窗提示
- 响应式布局适配全屏"
V2.0 时刻表接入统计页面
-整体UI风格及页面布局参考首页面
-添加两个卡片计划运行图接入线路数、实际运行图接入线路数；
-添加两个表格查询窗口计划图接入详情，实际运行图接入详情；
V3.0 实时报警统计页面
后端 API (/api/alarms):
   - 支持按线路筛选 (line_name)
   - 支持按日期筛选 (alarm_date)
   - 返回字段: id, line_name, alarm_content, time
前端功能:
   - 统计卡片：报警总数、今日报警
   - 筛选栏：线路筛选、日期筛选
   - 报警详情表：序号、线路名称、报警信息、报警时间
   - 分页支持
   - xcel下载


服务部署和启动
1. 启动后端服务
cd D:\Project\test1
python backend.py
后端将在 http://localhost:8000 运行
2. 启动前端服务
可以使用任意静态服务器，例如：
# Python内置服务器
python -m http.server 8080
然后访问 http://localhost:8080
3. 数据库配置
确保 pgsql-config.json 中的数据库配置正确：
{
  "host": "39.101.150.9",
